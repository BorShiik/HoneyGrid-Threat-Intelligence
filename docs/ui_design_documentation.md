# HoneyGrid Threat Intelligence — Документация проекта и UI/UX

## О проекте
**HoneyGrid** — это современная платформа класса Cyber Threat Intelligence (CTI) и Security Operations Center (SOC). Платформа предназначена для агрегации, корреляции и визуализации данных об атаках, поступающих с распределенной сети ханипотов (honeypots). 

Проект совмещает глубокую аналитику (профилирование атакующих, анализ вредоносных пейлоадов, STIX-индикаторы) с возможностями управления программно-конфигурируемыми сетями (SDN) для динамической маршрутизации вредоносного трафика.

## Философия дизайна: Premium Utility Design
Дизайн-система HoneyGrid построена на принципах **Premium Utility**:
1. **Отсутствие визуального шума**: Полный отказ от бессмысленной декоративности. Каждый пиксель интерфейса несет полезную нагрузку. Интерфейс должен спасать время аналитика в критических ситуациях (incident response).
2. **Высокая информационная плотность**: Компактные таблицы, моноширинные шрифты для данных и хешей, микро-чарты (sparklines) для отображения трендов прямо в карточках.
3. **Глубокая эстетика (Dark SOC Mode)**: 
   - Цветовая палитра основана на глубоком черном (`#09090b`) и оттенках графита (`zinc-900`).
   - Использование **Glassmorphism** (`glass-strong`): панели имеют легкую прозрачность, блюр фона (`backdrop-blur-xl`) и тончайшие полупрозрачные границы. Это создает ощущение "парящих" слоев (Z-depth) и живого интерфейса.
   - Цветные акценты используются строго для индикации статусов: Изумрудный (Normal/Online), Янтарный (Warning/System), Красный/Розовый (Critical/Malicious) и Синий (RDP/Network).
4. **Типографика**: 
   - **Inter** — для UI-элементов, заголовков и навигации (обеспечивает идеальную читаемость).
   - **JetBrains Mono** — для логов, IP-адресов, хешей, паролей и телеметрии (подчеркивает терминальную точность).
5. **Микро-анимации (Framer Motion)**: Плавные пружинные (spring) переходы при смене страниц, наведении (hover) и появлении новых данных в реальном времени. UI ощущается живым и отзывчивым.

## Глобальные компоненты (App Shell)
- **Боковая навигация (Sidebar)**: Сворачиваемая панель слева. При наведении плавно расширяется, открывая названия разделов. Активный пункт подсвечивается неоновым янтарным градиентом и маркером.
- **Верхняя панель (Topbar)**:
  - Хлебные крошки для понимания текущего контекста.
  - **Command Palette (`⌘K`)**: Глобальный поиск и навигация с клавиатуры для профессионалов.
  - **Кнопка Mute (Колокольчик)**: Быстрое отключение звуковых/визуальных уведомлений.
  - **Языковой переключатель**: Динамическая локализация (EN/PL/RU).
  - **Connection Pulse**: Индикатор соединения с SignalR (зеленый пульс = Online).
- **Attack Toaster**: Глобальный механизм всплывающих уведомлений в правом нижнем углу. Сообщает *только* о критических инцидентах (High/Critical) в реальном времени, предотвращая перегрузку аналитика.

## Описание страниц (Views)

### 1. Dashboard (`/`)
**Цель:** Сводка состояния инфраструктуры и ключевых метрик с первого взгляда.
- **KPI-карточки**: Общее количество атак, уникальные IP-источники, отфильтрованный трафик, критические угрозы. Внутри карточек — микро-графики (sparklines) для визуализации активности за последние 24 часа.
- **Sensor Breakdown**: Heatmap-полоса (прогресс-бар), показывающая распределение атак по типам ханипотов (SSH, Web, RDP).
- **Recent Threats**: Список последних классифицированных угроз с бейджами критичности.
- **Geo Distribution**: Топ стран-источников атак.

### 2. Threat Map (`/threat-map`)
**Цель:** Глобальная кибер-карта (Geo-spatial visualization).
- В центре расположен полностью интерактивный 3D-глобус (`react-globe.gl`).
- Каждая новая атака отображается в виде анимированной дуги (arc) от страны-источника к дата-центру HoneyGrid, а также точечным взрывом на карте.
- **Панели (Glass Overlays)**: Поверх карты наложены полупрозрачные плашки со статистикой (Total Attacks, Unique Sources) и списком активных источников в реальном времени.

### 3. Live Feed (`/live-feed`)
**Цель:** Сырой, терминальный поток логов в реальном времени.
- Потоковая таблица с высокой плотностью данных. Колонки: Время, Протокол, Источник (IP/Страна), Тип, Payload.
- Строки визуально подсвечиваются в зависимости от Severity (красный фон для Critical).
- Наличие панели управления: кнопка **Паузы (Pause)** для заморозки потока при изучении конкретного лога, фильтры по протоколам.

### 4. Threat Actors (`/actors`)
**Цель:** Разведка и профилирование (Attribution).
- Модуль группирует атаки и привязывает их к известным хакерским группировкам (APT, Syndicate, Script Kiddies).
- **Карточки акторов**: Показывают уровень угрозы (Threat Level), теги (TTPs - Tactics, Techniques, and Procedures) и описание поведения.
- Использование акцентных цветов для подчеркивания степени риска группировки.

### 5. SDN Monitoring (`/sdn`)
**Цель:** Мониторинг "железа" и маршрутизации.
- Витрина узлов сети (Core и Edge-серверы).
- Для каждого узла используются кастомные кольцевые графики (Circular Gauges) нагрузки на CPU и RAM.
- Отображение статусов (Online, Degraded, Offline) и тумблеров (Toggle) для включения динамической миграции (Dynamic Migration) маршрутов под DDOS-нагрузкой.

### 6. Captured Credentials (`/credentials`)
**Цель:** База скомпрометированных логинов и паролей.
- Отображает результаты брутфорс-атак на ханипоты.
- **Rank Bars**: Столбчатые диаграммы-рейтинги (Top Usernames, Top Passwords) прямо в виде списков с барами заполнения.
- Сводная таблица пар `Логин:Пароль` с указанием количества попыток подбора.

### 7. STIX / IoC (`/ioc`)
**Цель:** Экспорт индикаторов компрометации для SIEM-систем.
- Хранилище объектов STIX 2.1 (IP-адреса, хеши файлов, паттерны атак).
- Быстрые фильтры по типу объекта (Indicators, Attack Patterns, Threat Actors).
- Строка поиска и возможность экспортировать выбранные индикаторы в STIX Bundle формат (JSON) для автоматического импорта в защитные системы организации.

### 8. AI & Integrations (`/ai-integrations`)
**Цель:** Настройка автоматизации и работы ИИ-классификатора.
- Панели настройки моделей (например, подключение OpenAI/Azure ML для анализа payload'ов и определения TTPs).
- Управление ключами API, выбор модели, метрики точности и автоматических плейбуков реакции на инциденты.
